# 1. I/O Interfaces & Stream Processing

## Reader и Writer: Фундамент Go
Интерфейсы `io.Reader` и `io.Writer` — это гениальное решение в Go. Они позволяют абстрагироваться от источника данных (файл, сокет, буфер, тело HTTP запроса).

### Мощные утилиты
- `io.Copy`: Эффективное копирование данных из Reader в Writer. Использует внутренний буфер и старается применять системные оптимизации (например, `sendfile`).
- `io.TeeReader`: Читает данные и одновременно записывает их в Writer (полезно для логирования без изменения логики).
- `io.MultiReader`: Склеивает несколько Reader'ов в один последовательный поток.

## Stream Processing (Потоковая обработка)
Вместо того чтобы читать файл в память целиком (`os.ReadFile`), Senior-разработчик использует потоковое чтение (chunks). Это позволяет обрабатывать файлы любого размера при константном потреблении памяти.

## Вопросы для интервью (Senior/Staff)

### Почему Buffer в `io.Reader` передается снаружи (`Read([]byte)`)?
Это сделано для контроля аллокаций. Вызывающий код может создать один буфер и переиспользовать его в цикле, избегая давления на GC. Если бы `Read` сам возвращал `[]byte`, это вызывало бы аллокацию при каждом вызове.

### В чем разница между `bufio.Reader` и обычным `io.Reader`?
`bufio` добавляет промежуточный буфер в памяти. Вместо множества мелких системных вызовов `read()` (которые дороги), `bufio` читает данные большими кусками и отдает их приложению из памяти. Это критично при чтении из файлов и сокетов.

### Как работает `io.Pipe()`?
Он создает связанную пару (Reader и Writer). То, что записывается в Writer, мгновенно становится доступным для чтения из Reader. Это идеальный способ соединить две горутины по I/O без промежуточного слайса в памяти или временного файла.

### Что такое техника Zero-copy в I/O?
Это минимизация копирования данных между пространством ядра ОС и пространством приложения. В Go это достигается передачей указателей на буферы и использованием специализированных методов типа `ReadFrom` / `WriteTo`. Если Reader и Writer оба поддерживают эти интерфейсы, `io.Copy` может выполнить операцию на уровне ядра.
