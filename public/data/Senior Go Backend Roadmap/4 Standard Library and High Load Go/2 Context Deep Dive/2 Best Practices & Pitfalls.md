# 2. Best Practices & Pitfalls

## Использование Values в Контексте
Метод `WithValue` позволяет передавать данные сквозь слои.
**Staff правило**: Используйте это только для данных, которые не влияют на бизнес-логику (cross-cutting concerns).
- **ДА**: RequestID, TraceID, UserID (для логов), информация о локали.
- **НЕТ**: Аргументы функций, параметры запроса, конфигурация БД.

## Типичные ошибки (Pitfalls)
1. **Хранение контекста в структуре**: Контекст должен жить только в стеке вызовов. Хранение его в полях структуры (кроме редких случаев в драйверах) ведет к путанице и утечкам.
2. **Не вызывать Cancel**: Каждая функция `With...` возвращает `CancelFunc`. Её вызов очищает ресурсы таймеров и удаляет ссылку на ребенка из родительского списка.
3. **Использование контекста после отмены**: Все функции, принимающие контекст, должны проверять `ctx.Err()` перед выполнением тяжелой работы.

## Вопросы для интервью (Senior/Staff)

### Почему ключи в `WithValue` должны быть кастомного типа?
Чтобы избежать коллизий. Если два пакета используют строку `"token"`, они могут перезаписать данные друг друга. Использование неэкспортируемого типа (`type ctxKey int`) гарантирует, что только ваш пакет сможет прочитать и записать это значение.

### Как проверить, отменен ли контекст, не блокируясь на канале `Done()`?
Использовать неблокирующий `select`:
```go
select {
case <-ctx.Done():
    return ctx.Err()
default:
}
```
Или просто вызвать `ctx.Err() != nil`. Но `select` чаще используется в циклах обработки данных.

### Влияет ли количество значений в контексте на производительность?
Да. Поиск по `Value` — это рекурсивный обход дерева вверх до корня. Сложность **O(D)**, где D — глубина дерева контекста. Если у вас сотни ключей, поиск станет медленным. Контекст — это не мапа!

### Как реализовать "Graceful" отмену в сложной цепочке горутин?
Нужно передавать один и тот же `ctx` во все горутины и использовать `errgroup.WithContext`. Если одна горутина упадет с ошибкой, `errgroup` отменит контекст для всех остальных, обеспечивая чистую остановку всей системы.
