# 1. Kubernetes Essentials for Go Devs

## Основные сущности K8s
Senior разработчик должен понимать, куда деплоится его код:
1. **Pod**: Минимальная единица. Группа из одного или нескольких контейнеров.
2. **Deployment**: Описание того, как разворачивать Поды (количество реплик, стратегия обновления).
3. **Service**: Стабильный IP/DNS для группы Подов. Балансировщик внутри кластера.
4. **ConfigMap / Secret**: Способ прокинуть конфигурацию и пароли в приложение без пересборки образа.

## Вопросы для интервью (Senior/Staff)

### Как приложение узнает, что оно работает в Kubernetes?
Через переменные окружения (Environment Variables) или монтируемые файлы. Также можно использовать **Downward API**, чтобы прокинуть в приложение имя Пода, IP ноды или лимиты ресурсов.

### В чем разница между ClusterIP и NodePort?
- **ClusterIP**: Сервис доступен только внутри кластера. (Безопасно, используется для связи микросервисов).
- **NodePort**: Пробрасывает порт сервиса на каждую ноду кластера. (Позволяет достучаться извне, но менее гибко).
Для внешнего доступа обычно используют **Ingress**.

### Как реализовать Graceful Shutdown в Go под управлением Kubernetes?
Когда K8s хочет убить Под, он посылает сигнал `SIGTERM`. Приложение должно:
1. Перестать принимать новые запросы (Readiness probe должна начать отвечать "не готов").
2. Дождаться завершения активных запросов (HTTP/gRPC/Workers).
3. Закрыть соединения с БД.
4. Завершить процесс.
На это дается время `terminationGracePeriodSeconds` (по умолчанию 30с).

### Что такое "Sidecar" контейнер?
Дополнительный контейнер в том же Поде, который помогает основному.
*Примеры*: Лог-ротатор, прокси для Service Mesh (Istio Envoy), агент для сбора метрик. Они делят одну сеть (localhost) и одни тома (Volumes).
