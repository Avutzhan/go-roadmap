# 2. Design Patterns & Functional Options

## Паттерны GoF в Go реальности
Многие классические паттерны в Go реализуются иначе или не нужны вовсе благодаря функциям первого класса и интерфейсам.

### Популярные паттерны в Go:
1. **Singleton**: Реализуется через пакетные переменные и `sync.Once`.
2. **Strategy**: Легко реализуется передачей функции или интерфейса в качестве аргумента.
3. **Decorator**: Оборачивание одного типа в другой (популярно в middleware и логировании).
4. **Adapter**: Создание структуры, которая удовлетворяет нужному интерфейсу, используя методы другой структуры.

## Functional Options Pattern
Это стандарт де-факто в Go для настройки сложных объектов (например, HTTP серверов или клиентов БД).
**Проблема**: Конструкторы с 10 аргументами неудобны. Структуры конфигураций (`Config`) неудобны для дефолтных значений.
**Решение**: Функции, которые принимают указатель на объект и меняют его поля.

```go
func WithTimeout(t time.Duration) Option {
    return func(s *Server) {
        s.timeout = t
    }
}

s := NewServer(WithTimeout(5 * time.Second))
```

## Вопросы для интервью (Senior/Staff)

### Почему паттерн Factory Method часто считается избыточным в Go?
В Go функции могут возвращать интерфейсы. Обычная функция `New...` уже является фабрикой. Сложные иерархии фабрик (Abstract Factory) в Go встречаются редко, так как композиция и интерфейсы позволяют достичь той же гибкости меньшей кровью.

### В чем преимущества Functional Options перед передачей структуры Config?
1. **API stability**: Вы можете добавить новую опцию, не ломая обратную совместимость существующих вызовов `NewServer`.
2. **Readability**: Код инициализации читается как документация.
3. **Zero-values**: Вам не нужно проверять, какое поле в `Config` было заполнено, а какое — нет. Опции применяются только если они переданы.

### Как реализовать многопоточный Singleton без Race Conditions?
Использовать `sync.Once`. Это гарантирует, что инициализация выполнится ровно один раз, и все последующие обращения будут ждать завершения первого (если оно еще идет).

### Когда паттерн Builder оправдан в Go?
Когда объект создается пошагово (например, построение сложного SQL запроса или цепочки фильтров). Для простых структур `Functional Options` обычно предпочтительнее.
