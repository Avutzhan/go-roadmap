# 1. Trees & Graphs

## Деревья (Trees)
- **DFS (Depth First Search)**: Обход в глубину. Бывает Pre-order, In-order (дает сортировку в BST), Post-order.
- **BFS (Breadth First Search)**: Обход в ширину (по уровням). Использует очередь.
- **BST (Binary Search Tree)**: Позволяет искать, вставлять и удалять за **O(log N)** в среднем.

## Графы (Graphs)
- **Представление**: Adjacency List (список смежности) — в Go это `map[int][]int`.
- **Dijkstra**: Поиск кратчайшего пути во взвешенном графе.
- **Topological Sort**: Упорядочивание узлов с учетом зависимостей (например, порядок сборки пакетов или выполнения задач).

## Вопросы для интервью (Senior/Staff)

### Как проверить, является ли бинарное дерево деревом поиска (BST)?
Недостаточно просто проверить, что `left < root < right` для каждого узла. Нужно передавать диапазон `[min, max]` сверху вниз и проверять, что каждый узел попадает в свои границы. Это классическая задача на рекурсию с передачей состояния.

### В чем разница между DFS и BFS при поиске кратчайшего пути?
BFS всегда находит кратчайший путь в невзвешенном графе (по количеству ребер). DFS этого не гарантирует, но потребляет меньше памяти, если граф очень широкий.

### Что такое "Strongly Connected Components" (сильно связанные компоненты)?
Группы узлов в графе, где из любого узла группы можно добраться до любого другого. Используется в анализе социальных сетей или поиске циклов зависимостей в архитектуре.

### Для чего используется Trie (Префиксное дерево) в промышленном коде?
Для быстрой фильтрации по префиксу (автодополнение текста, роутинг HTTP запросов по путям). Поиск в Trie зависит от длины слова **O(L)**, а не от количества слов в словаре, что делает его экстремально быстрым для роутинга миллионов эндпоинтов.
