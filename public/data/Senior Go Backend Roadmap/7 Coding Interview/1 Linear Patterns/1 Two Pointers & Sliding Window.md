# 1. Two Pointers & Sliding Window

## Техника двух указателей (Two Pointers)
Используется для обработки отсортированных массивов или связных списков с минимальной памятью **O(1)**.
- **Сценарии**: 
    - Поиск пары с заданной суммой (указатели на концах, сдвигаем к центру).
    - Удаление дубликатов на месте (один указатель "быстрый", другой "медленный").
    - Разворот строки/массива.
    - Проверка палиндрома.

## Скользящее окно (Sliding Window)
Используется для поиска непрерывного подмассива или подстроки, удовлетворяющих условию.
- **Сценарии**:
    - Максимальная сумма подмассива длины K (фиксированное окно).
    - Самая длинная подстрока без повторяющихся символов (динамическое окно).
    - Минимальная длина подмассива с суммой >= S.

## Вопросы для интервью (Senior/Staff)

### В чем преимущество Sliding Window перед вложенными циклами?
Вложенные циклы обычно дают сложность **O(N²)**. Скользящее окно проходит по массиву один раз двумя указателями, что дает линейную сложность **O(N)**. Для Senior-позиции важно уметь доказывать амортизированную стоимость этого алгоритма.

### Как найти "Средний элемент" связного списка за один проход?
Использовать технику "Заяц и Черепаха" (Fast and Slow Pointers). Быстрый указатель идет через два узла, медленный — через один. Когда быстрый дойдет до конца, медленный будет ровно посередине. Эта же техника используется для поиска цикла в списке.

### Как Sliding Window помогает в реализации Rate Limiting?
Алгоритм **Sliding Window Log** позволяет хранить метки времени всех запросов за последний период. При новом запросе мы удаляем старые метки (те, что выпали из окна) и проверяем количество оставшихся. Это дает идеальную точность, в отличие от Fixed Window, но требует больше памяти.

### Как решить задачу "3Sum" (найти все уникальные тройки с суммой 0) эффективно?
Нужно отсортировать массив, зафиксировать один элемент в цикле и применить технику двух указателей для поиска остальных двух в оставшейся части. Сложность составит **O(N²)**, что на порядок лучше перебора **O(N³)**.
