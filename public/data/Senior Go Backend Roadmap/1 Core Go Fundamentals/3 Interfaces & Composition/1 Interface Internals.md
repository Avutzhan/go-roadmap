# 1. Interface Internals: iface & eface

## Основные структуры
В рантайме Go интерфейсы представлены двумя структурами (runtime/runtime2.go):

### 1. eface (Empty Interface)
Используется для `interface{}` или `any`.
```go
type eface struct {
    _type *_type         // Информация о типе
    data  unsafe.Pointer // Указатель на данные
}
```

### 2. iface (Interface with methods)
Используется для интерфейсов с объявленными методами.
```go
type iface struct {
    tab  *itab          // Таблица методов и информация о типе
    data unsafe.Pointer // Указатель на данные
}
```

## Устройство itab и динамическая диспетчеризация
`itab` содержит:
- **inter**: описание самого интерфейса.
- **_type**: информация о конкретном типе.
- **hash**: копия хеша типа для быстрой проверки при type assertion.
- **fun**: массив указателей на методы. Вызов метода через интерфейс требует двойного разыменования указателя, что исключает возможность инлайнинга компилятором.

## Хранение данных в интерфейсе (Value Copying)
При присваивании значения интерфейсу Go всегда делает копию данных:
- Если значение маленькое (помещается в указатель), оно копируется напрямую.
- Если значение большое или требует выделения памяти (escape к heap), интерфейс хранит указатель на копию данных.
Это важно учитывать при работе с большими структурами в высокопроизводительных системах.

## Вопросы для интервью (Senior/Staff)

### Typed Nil vs Interface Nil (Кейс с error)
Интерфейс считается `nil` только если и тип, и данные равны `nil`. Это частая причина багов в обработке ошибок:
```go
var err *MyError = nil
var i error = err
fmt.Println(i == nil) // false
```
Функция вернет "ошибку", которая не равна `nil`, хотя внутри лежит `nil` указатель. Проверку на `nil` нужно делать до упаковки в интерфейс.

### Стоимость Type Assertion
Type assertion — это рантайм-проверка. Рантайм сравнивает хеши типов или проходит по иерархии типов (в случае сложных вложений). Это накладная операция по сравнению со статическим вызовом.

### Почему any — это просто eface?
`any` — это синтаксический сахар (alias), введенный в 1.18. На уровне рантайма это `eface`, которая не имеет таблицы методов (`itab`), поэтому через нее нельзя вызвать ни один метод без приведения к типу.
