
# 🏷 6. Struct tags

## 📌 Что это

Метаданные для структур (например JSON)

```go
type User struct {
    Name string `json:"name"`
    Age  int    `json:"age"`
}
```

---

## 📦 Использование

```go
json.Marshal(user)
```

👉 Go понимает как назвать поля

---

## 🧠 Аналогия

Python (dataclasses / pydantic):

```python
class User(BaseModel):
    name: str
```

---

## 💡 Зачем нужно

* API
* JSON
* базы данных (ORM)

---

# 🧠 Итог (простыми словами)

| Тема        | Что это                                 |
| ----------- | --------------------------------------- |
| shadowing   | можно случайно создать новую переменную |
| iota        | авто-нумерация констант                 |
| range       | работает с копиями                      |
| switch      | проверка типов                          |
| init        | код до main                             |
| zero value  | значения по умолчанию                   |
| struct tags | как сериализовать структуру             |

---

# 🏷 6. Struct Tags (Senior)

## 📌 Это reflection

```go
type User struct {
    Name string `json:"name"`
}
```

👉 работает через `reflect`

---

## ⚠️ Цена

* reflection = медленно
* не type-safe

---

## 🔥 Production нюансы

### omitempty

```go
Age int `json:"age,omitempty"`
```

👉 0 → не отправится

👉 иногда это баг

---

### скрытие поля

```go
Password string `json:"-"`
```

---

## 📌 Валидация / ORM

```go
validate:"required"
gorm:"primaryKey"
```

---

## 💬 Интервью

> "Struct tags — это декларативный способ описания поведения, но они зависят от reflection, поэтому я учитываю их влияние на performance и читаемость."

---

# 🧠 Итог (уровень Senior)

Ты должен мыслить так:

| Тема        | Senior thinking           |
| ----------- | ------------------------- |
| shadowing   | источник багов → избегать |
| iota        | типизация + читаемость    |
| range       | копии + goroutine bugs    |
| init        | избегать магии            |
| zero value  | использовать осознанно    |
| struct tags | reflection + API влияние  |
