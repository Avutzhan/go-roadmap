# 🔥 Шаг 1: Bit flags (разберём до идеала)

Это самый главный пробел. Если ты это поймёшь — половина темы закроется.

---

## 📌 Сначала база: что такое бит

Число в памяти — это биты:

```
1  = 00000001
2  = 00000010
4  = 00000100
8  = 00001000
```

👉 Каждое число — это **включённый один бит**

---

## 📌 Теперь iota + сдвиг

```go
const (
    Read = 1 << iota   // 1 << 0 = 1
    Write              // 1 << 1 = 2
    Execute            // 1 << 2 = 4
)
```

Получаем:

```
Read    = 00000001
Write   = 00000010
Execute = 00000100
```

---

## 📌 Теперь магия: объединение прав

```go
perms := Read | Write
```

👉 `|` = побитовое ИЛИ

```
Read   = 00000001
Write  = 00000010
----------------
Result = 00000011
```

👉 У тебя теперь **2 права в одном числе**

---

## 📌 Проверка права

```go
perms & Execute
```

👉 `&` = побитовое И

```
perms   = 00000011
Execute = 00000100
------------------
Result  = 00000000 → нет права
```

---

```go
perms & Write
```

```
00000011
00000010
--------
00000010 → есть право
```

---

## 🧠 Как объяснить на собесе (золото)

> "Я использую `1 << iota` чтобы задать уникальные биты для каждого permission. Это позволяет хранить несколько прав в одном числе через bitwise OR и проверять их через AND."

---

## ⚠️ Почему это круто (важно!)

Вместо:

```go
[]string{"read", "write"}
```

Ты используешь:

```go
uint8
```

👉 быстрее
👉 меньше памяти
👉 удобно проверять
