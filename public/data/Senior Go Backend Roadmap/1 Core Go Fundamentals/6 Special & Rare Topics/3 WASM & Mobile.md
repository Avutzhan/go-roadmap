# 3. WASM & Mobile

## WebAssembly (WASM)
Go поддерживает компиляцию под WASM (`GOOS=js GOARCH=wasm`). Это позволяет запускать Go-код в браузерах.

### Ограничения WASM (WASM Limitations)
Для Staff уровня важно знать:
1. **Single-threaded**: В текущей реализации Go для браузерного WASM рантайм работает в однопоточном режиме. Горутины существуют, но они не выполняются параллельно на разных ядрах процессора (используется кооперативная многозадачность в рамках одного потока JS).
2. **Бинарный размер**: Минимальный файл весит ~2-5 МБ из-за встроенного рантайма и GC. Это делает Go менее привлекательным для мелких фронтенд-скриптов по сравнению с Rust или AssemblyScript.

## Go Mobile
Инструментарий `gomobile` позволяет использовать Go в мобильной разработке через создание библиотек (`.aar`, `.framework`) с общим бизнес-ядром (Shared Core).

## Вопросы для интервью (Senior/Staff)

### Почему Go в WASM работает медленнее, чем нативный код?
Из-за необходимости прослойки между рантаймом Go и движком JS/WASM (системные вызовы через JavaScript). Также на производительность влияет реализация Garbage Collector, которая не так оптимальна в среде WASM, как в нативной ОС.

### Каков основной кейс использования WASM на Go?
Перенос сложной существующей логики (например, валидация бинарных форматов, криптографические алгоритмы) из бэкенда на фронтенд без необходимости переписывать код на JavaScript/TypeScript.
