# 1. Slices: Under the Hood

## 📌 Что это
Слайс — это дескриптор (header), описывающий сегмент нижележащего массива.

В рантайме Go (runtime/slice.go) это структура:
```go
type slice struct {
    array unsafe.Pointer // Указатель на массив
    len   int            // Длина
    cap   int            // Емкость
}
```

---

## 💡 Важные концепции (Senior Level)

### 1. Формула роста (Threshold 256)
При достижении предела емкости происходит аллокация нового массива.
- Если cap < 256: емкость удваивается (x2).
- Если cap >= 256: используется формула `newcap += (newcap + 3*256) / 4`. Это обеспечивает плавный переход от x2 к x1.25.

### 2. Slice Aliasing и Shared Arrays
Слайсы, полученные из одного массива, разделяют его память. Изменение элемента в одном слайсе затронет все остальные слайсы, указывающие на этот же участок памяти.
```go
a := []int{1, 2, 3}
b := a[:2]
b[0] = 100 // Изменит и a[0], так как указатель на массив один.
```

### 3. Механика append: Realloc vs Copy
- Если `len + 1 <= cap`, `append` просто записывает значение в существующий массив.
- Если `len + 1 > cap`, создается новый массив, данные копируются, и указатель в хедере обновляется.
Это поведение делает результат `append` недетерминированным для внешних ссылок на тот же массив (алиасинг может пропасть после реаллокации).

### 4. Nil vs Empty Slice
- `var s []int`: Слайс со значением `nil`. Хедер полностью нулевой (array=0, len=0, cap=0).
- `s := []int{}`: Пустой слайс. Хедер имеет len=0, cap=0, но `array` указывает на специальный `zerobase` в рантайме.
При сериализации в JSON: `nil` -> `null`, `empty` -> `[]`.

### 5. Трех-индексный слайсинг (Full slice expression)
`s := source[low:high:max]`
Позволяет жестко ограничить `cap` нового слайса. Это гарантирует, что любой последующий `append` в `s` вызовет аллокацию и не повредит данные в `source`.

---

## 🔥 Сценарии в Production
**Утечки памяти при слайсинге больших массивов.**
Выражение `tiny := big[0:1]` сохраняет ссылку на весь массив байт исходного `big`. Весь объем памяти не будет освобожден GC, пока жив `tiny`.
**Решение:** использовать `copy()` в новый слайс или `slices.Clone()`.

## 💬 Как говорить на интервью
"Слайс — это легковесный хедер над массивом. При работе с ними важно учитывать эффекты разделяемой памяти (aliasing) и следить за тем, чтобы не удерживать ссылки на огромные массивы через маленькие слайсы (утечки памяти). Для предотвращения нежелательной модификации исходных данных в критических местах я использую full slice expressions."
