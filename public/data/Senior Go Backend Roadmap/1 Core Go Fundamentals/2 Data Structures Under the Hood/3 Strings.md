# 3. Strings: Under the Hood

## 📌 Что это
Строка в Go — это неизменяемый (read-only) слайс байт.

Структура в рантайме:
```go
type stringStruct struct {
    str unsafe.Pointer // Указатель на массив байт
    len int            // Длина в байтах
}
```

---

## 💡 Важные концепции (Senior Level)

### 1. Immutability и Memory Sharing (Substrings)
Строки неизменяемы. Слайсинг строки не копирует данные, а создает новый хедер, указывающий на ту же область памяти.
```go
s := "huge string..."
sub := s[:5] // sub делит ту же память, что и s.
```
**Риск:** Если `s` — это длинный ответ от API, а `sub` — маленькое поле из него, то весь объем `s` будет висеть в памяти, пока жив `sub`.

### 2. Конвертация string <-> []byte (Performance)
- Стандартное преобразование `string(bytes)` всегда выполняет копирование данных для обеспечения иммутабельности.
- **Zero-copy:** В Go 1.20+ через `unsafe.StringData` и `unsafe.String` можно выполнять преобразования без аллокаций, если гарантируется, что исходный слайс байт не будет изменен.

### 3. UTF-8 и итерация
- `len(s)` возвращает количество байт.
- Индексация `s[i]` обращается к байту.
- `for range s` выполняет декодирование UTF-8 и возвращает `rune` (int32). Это дороже по процессору, чем итерация по байтам.

### 4. Строки как ключи мап
Хеширование длинных строк может быть накладным. Рантайм оптимизирует поиск, сравнивая сначала длину строк и их `tophash` перед полным посимвольным сравнением.

---

## 🔥 Сценарии в Production
**Эффективная конкатенация.**
При сборке строки из частей в цикле использование `+=` создает квадратичную сложность по времени и памяти. Для этих целей используется `strings.Builder`, который работает с расширяемым буфером и позволяет сделать `Grow()` для преаллокации.

## 💬 Как говорить на интервью
"Строки в Go — это иммутабельные дескрипторы поверх массивов байт. Я учитываю специфику слайсинга строк, чтобы избежать утечек памяти (memory sharing), и использую strings.Builder для эффективной сборки. При работе с Unicode важно помнить, что индексация дает доступ к байту, а не к символу."
