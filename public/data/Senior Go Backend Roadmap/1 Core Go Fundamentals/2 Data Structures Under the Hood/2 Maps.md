# 2. Maps: Under the Hood

## 📌 Что это
Map в Go — это хеш-таблица на основе бакетов. Структура в рантайме — `hmap`.

Ключевые составные части:
- **Buckets**: массив структур `bmap`. Каждый бакет хранит до 8 пар ключ-значение.
- **Tophash**: массив из 8 бит в каждом бакете для быстрой фильтрации ключей при поиске.
- **Overflow buckets**: связные списки бакетов, которые создаются, когда основной бакет переполнен.

---

## 💡 Важные концепции (Senior Level)

### 1. Load Factor и Эвакуация
Когда количество элементов превышает среднюю нагрузку (Load Factor > 6.5), мапа начинает рост.
- Создается новый массив бакетов в 2 раза больше.
- Перенос данных (эвакуация) происходит постепенно: при каждой операции записи или удаления переносятся один или два бакета. Это размывает стоимость роста по времени.

### 2. Потокобезопасность и Concurrency
- **Concurrent Reads**: Безопасны. Несколько горутин могут читать одну мапу одновременно без блокировок.
- **Concurrent Writes**: Запрещены. Если рантайм обнаруживает одновременную запись (или чтение во время записи), происходит фатальная ошибка: `concurrent map writes`. Для защиты требуется внешняя синхронизация (Mutex) или типы пакет `sync`.

### 3. sync.Map vs map + Mutex
- `map + Mutex/RWMutex`: Оптимальный выбор для большинства случаев.
- `sync.Map`: Оптимизирована для двух сценариев:
    1. Постоянное чтение и редкая запись (read-mostly).
    2. Когда несколько горутин работают с непересекающимися наборами ключей (разные ключи в разных потоках).
В остальных случаях обычная мапа с мьютексом быстрее.

### 4. Деградация и Коллизии
При плохой хеш-функции время поиска может деградировать до `O(n)`. Рантайм Go использует рандомизированную при каждом запуске хеш-функцию, чтобы предотвратить атаки типа Hash DoS.

---

## 🔥 Сценарии в Production
**Мапа не возвращает память ОС.**
При удалении ключей через `delete()` бакеты не освобождаются и не схлопываются. Если мапа разово выросла до огромных размеров, она будет занимать этот объем памяти до завершения программы.
**Решение:** Если мапа используется как временный буфер огромных размеров, после обработки данных ее нужно обнулить (`m = nil`) или пересоздать.

## 💬 Как говорить на интервью
"Мапа — это hmap с инкрементальной эвакуацией бакетов. Я использую `make(map[K]V, hint)` для минимизации аллокаций при росте. При выборе между Mutex и sync.Map я отталкиваюсь от профиля нагрузки: для тяжелой записи обычный Mutex эффективнее за счет отсутствия оверхеда на интерфейсы."
