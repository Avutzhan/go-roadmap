# 🎯 Мини-советы по подготовке (Senior Level)

---

## 1. 🧩 Algorithms (минимум, но качественно)

* Делай **3–4 задачи в день максимум**
* Фокус на:

  * HashMap
  * Sliding Window
  * BFS / DFS
  * Heap / Priority Queue
* Если не решил за **30–40 минут → смотри разбор**
* Обязательно **реализуй сам после просмотра**

👉 Цель: **не зафейлиться, а не стать чемпионом LeetCode**

---

## 2. 🧠 System Design (ключевая зона)

Не просто читай — **думай и объясняй**

Формат каждого кейса:

```
1. Requirements
2. Constraints (scale, latency)
3. High-level design
4. Deep dive
5. Trade-offs
6. Failure scenarios
```

👉 Ты должен уметь:

* накидать архитектуру за 5 минут
* объяснить её на английском
* защитить решение

---

## 3. ⚙️ Go Core (говори как инженер)

Не просто “знаю”, а:

👉 “я это видел в проде”

Проговаривай вслух:

* GMP scheduler
* channels vs mutex
* GC behavior
* race conditions

👉 добавляй:

* trade-offs
* реальные кейсы

---

## 4. 🎤 English + Communication (критично)

Каждый день:

* объясняй system design **вслух**
* отвечай на вопросы **как на интервью**

Используй фразы:

* “From a scalability perspective…”
* “The trade-offs here are…”
* “One potential issue is…”
* “To mitigate this…”

👉 важно не только знать, но **звучать уверенно**

---

## 5. 🧠 STAR Method (обязательно)

Подготовь **5 сильных историй**:

* сложный баг
* high-load система
* оптимизация
* конфликт в команде
* архитектурное решение

Формат:

```
Situation → Task → Action → Result
```

---

## 6. ⚖️ Trade-offs mindset (везде)

Любой ответ:

```
Pros → Cons → Decision → Risks
```

Пример:

> Redis fast, but introduces eventual consistency and memory constraints.

---

## 7. 💣 Production Thinking (что отличает Senior)

Всегда думай:

* что сломается?
* где bottleneck?
* как масштабировать?
* что при high load?
