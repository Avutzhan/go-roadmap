# 2. Composite & Covering Indexes

## Составной индекс (Composite Index)
Индекс по нескольким колонкам одновременно: `INDEX (last_name, first_name)`.

### Правило левого префикса (Selective Prefix)
Индекс будет работать, если в `WHERE` указаны:
- `last_name`
- `last_name` И `first_name`
Он **не будет** работать, если указан только `first_name`.

## Покрывающий индекс (Covering Index)
Ситуация, когда все данные, запрашиваемые в `SELECT`, уже есть в самом индексе.
*Пример*: `SELECT age FROM users WHERE last_name = 'Smith'`. Если есть индекс `(last_name, age)`, БД не будет лежать в основную таблицу (Heap) за данными.
**Результат**: **Index Only Scan** — экстремально быстро.

## Вопросы для интервью (Senior/Staff)

### Как порядок колонок в составном индексе влияет на его селективность?
Нужно ставить вперед ту колонку, которая сильнее всего фильтрует данные (имеет больше уникальных значений). Если вы поставите первым `gender`, индекс разделит таблицу всего на две части, что мало поможет поиску.

### Что такое Partial Index (Частичный индекс)?
Индекс, который строится только по части строк: `CREATE INDEX ... WHERE active = true`. 
**Плюс**: Занимает гораздо меньше места и быстрее обновляется. Идеально для флага "удален" (is_deleted) или обработки "горячих" данных.

### В чем разница между Clustered и Non-Clustered индексами?
- **Clustered index**: Сама таблица отсортирована физически по этому индексу (обычно Primary Key в MySQL/InnoDB). Может быть только один на таблицу.
- **Non-clustered**: Отдельная структура, хранящая указатели на строки в куче (Postgres).

### Как избежать Seq Scan, если индекс есть, но БД его не использует?
1. Проверить статистику (`ANALYZE`).
2. Убедиться, что не используется функция над колонкой: `WHERE YEAR(date) = 2023` отключает индекс. Нужно: `WHERE date >= '2023-01-01'`.
3. Проверить типы данных (неявное приведение типов отключает индекс).
