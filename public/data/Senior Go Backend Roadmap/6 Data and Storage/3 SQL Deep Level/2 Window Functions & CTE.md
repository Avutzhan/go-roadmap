# 2. Window Functions & CTE

## Window Functions (Оконные функции)
Выполняют вычисления над набором строк, связанных с текущей строкой, не сворачивая их в одну (в отличие от `GROUP BY`).
- **Синтаксис**: `FUNCTION() OVER (PARTITION BY ... ORDER BY ...)`.
- **ROW_NUMBER()**: Порядковый номер (1, 2, 3).
- **RANK()**: Ранг с пропусками при дублях (1, 2, 2, 4).
- **DENSE_RANK()**: Ранг без пропусков (1, 2, 2, 3).
- **LAG / LEAD**: Доступ к предыдущей или следующей строке (удобно для расчета дельты во времени).

## CTE (Common Table Expressions)
Конструкция `WITH`. Позволяет создавать временные именованные наборы данных.
- **Recursive CTE**: Позволяет делать рекурсивные запросы. Идеально для обхода деревьев (категории, иерархия сотрудников, графы).

## Вопросы для интервью (Senior/Staff)

### Как с помощью оконной функции найти топ-3 зарплаты в каждом отделе?
```sql
SELECT * FROM (
  SELECT name, salary, department,
         DENSE_RANK() OVER (PARTITION BY department ORDER BY salary DESC) as r
  FROM employees
) as t WHERE r <= 3;
```
Это классический пример, который Senior должен писать без запинки.

### В чем преимущество CTE перед вложенными запросами (Subqueries)?
1. **Читаемость**: CTE структурирует запрос сверху вниз, а не "изнутри наружу".
2. **Reusability**: К одному CTE можно обращаться несколько раз внутри одного запроса.
3. **Рекурсия**: Только CTE позволяют выполнять рекурсивные операции.

### Когда оконные функции могут стать проблемой для производительности?
Когда окно (Partition) слишком большое и не покрыто индексами. БД приходится делать тяжелый `Sort` в памяти или на диске (External Sort), что критично для High-load.

### Как рассчитать "Retention" или "Churn Rate" средствами SQL?
Через функции `LAG` или `LEAD`, сопоставляя дату текущего входа пользователя с датой его предыдущего входа. Это избавляет от необходимости выкачивать миллионы строк в Go и считать это в памяти.
