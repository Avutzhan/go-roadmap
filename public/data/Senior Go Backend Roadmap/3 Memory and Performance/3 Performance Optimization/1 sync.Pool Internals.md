# 1. sync.Pool Internals

## Предназначение
`sync.Pool` используется для кэширования и переиспользования выделенных объектов, чтобы избежать аллокаций в куче и снизить нагрузку на Garbage Collector. Это идеальный инструмент для буферов байт, структур запросов и тяжелых объектов.

## Внутреннее устройство (Staff уровень)
`sync.Pool` устроен сложнее, чем просто мапа или очередь под мьютексом:
- **Local Pools**: Каждому P (процессору) соответствует свой локальный пул (`poolLocal`). Это позволяет горутинам брать объекты без конкуренции (LIFO/FIFO очереди внутри P).
- **Victim Cache**: Начиная с Go 1.13, объекты в пуле не удаляются мгновенно всеми сразу при GC. Есть "жертвенный кэш" (victim cache), который сохраняет объекты на один лишний цикл GC, сглаживая просадки производительности.

## Вопросы для интервью (Senior/Staff)

### Почему в sync.Pool нельзя хранить данные, критичные к потере?
Потому что пул не дает никаких гарантий хранения. Рантайм может очистить его в любой момент во время Garbage Collection. Пул — это только оптимизация памяти, а не кэш данных.

### Как предотвратить утечку памяти через sync.Pool?
Если вы кладете в пул огромные слайсы байт (например, после обработки 10MB файла), они будут занимать память вечно. Решение: либо проверять размер объекта перед возвращением в пул (`if cap(b) > maxAllowed { return }`), либо использовать несколько пулов для разных классов размеров (size classes).

### Пул под капотом использует мьютекс или атомики?
В основном пути (Fast Path) при обращении к локальному кэшу процессора (P) используются атомики и локальные очереди (без блокировок). Если локальный пул пуст, поток идет воровать объекты у других P или в глобальный кэш — там уже могут использоваться блокировки. Это делает `sync.Pool` чрезвычайно эффективным на многоядерных системах.

### В чем разница между New() и Get()?
- `Get()`: Пытается найти свободный объект. Если пул пуст, он вызывает функцию `New`, которую вы определили при создании пула.
- При получении объекта через `Get()` обязательно нужно сбросить его состояние (очистить слайс, обнулить поля), прежде чем использовать, так как объект может содержать данные от прошлого владельца.
