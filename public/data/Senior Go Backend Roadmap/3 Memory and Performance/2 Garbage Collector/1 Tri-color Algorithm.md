# 1. Tri-color Algorithm: Internals

## Суть алгоритма Mark & Sweep
Garbage Collector в Go является конкурентным (запускается параллельно с программой) и не перемещает объекты в памяти (non-moving).

### Три цвета (Три состояния разметки)
1. **Белый (White)**: Объекты, которые GC еще не видел. В конце цикла все белые объекты удаляются.
2. **Серый (Grey)**: Объекты, до которых GC дотянулся, но еще не проверил их внутренние ссылки (поля/указатели).
3. **Черный (Black)**: Объекты, которые точно "живы", и все их ссылки уже проверены и помечены серыми.

### Процесс (Цикл)
- Начинается с **Roots** (глобальные переменные, стеки горутин). Все объекты, на которые они указывают, становятся серыми.
- GC берет серый объект, помечает его черным, а все объекты, на которые он ссылается — серыми.
- Цикл повторяется, пока не закончатся серые объекты.

## Вопросы для интервью (Senior/Staff)

### Почему Go GC не "схлопывает" (не дефрагментирует) кучу?
Это сознательный выбор в пользу низкой задержки (Low Latency). Перемещение объектов требовало бы остановки программы (STW) на долгое время для обновления всех указателей. Вместо дефрагментации Go использует сложные списки свободных блоков разного размера, что минимизирует фрагментацию без перемещения данных.

### Что такое "Write Barrier" и когда он включается?
Запись (Write Barrier) — это логика, которая включается только во время фазы разметки GC. Если программа пытается записать указатель в черный объект, барьер перехватывает это действие и помечает целевой объект серым. Это необходимо, чтобы GC случайно не удалил объект, на который внезапно появилась ссылка в уже "проверенной" части графа.

### Каков оверхед от работы GC на CPU?
Рантайм стремится использовать не более 25% ресурсов CPU для работы GC (`GOMAXPROCS * 0.25`). Если горутины выделяют память слишком быстро, рантайм может заставить их помогать GC в разметке (механизм **GC Assist**), что замедляет выполнение бизнес-логики.

### Влияние количества горутин на фазу Mark Termination
Хотя основная работа GC идет конкурентно, финальная фаза требует кратковременной остановки всех потоков (STW) для сканирования стеков. Если у вас миллионы горутин, время этой паузы может вырасти, так как планировщику нужно дождаться безопасной точки остановки для каждой из них.
