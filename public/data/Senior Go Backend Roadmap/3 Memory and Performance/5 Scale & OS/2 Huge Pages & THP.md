# 2. Huge Pages & THP

## Что такое Huge Pages?
Стандартный размер страницы памяти в Linux — 4KB. Huge Pages позволяют использовать страницы по 2MB или 1GB.
- **Цель**: Сократить размер таблицы TLB (Translation Lookaside Buffer) в CPU. TLB — это быстрый кэш для преобразования виртуальных адресов в физические. Чем меньше страниц, тем меньше промахов TLB.

## Huge Pages в Go
Go — это приложение, интенсивно работающее с памятью. Использование Huge Pages может ускорить сервис на 5-10%.

## Вопросы для интервью (Senior/Staff)

### В чем опасность Transparent Huge Pages (THP) для Go?
THP — это механизм ядра Linux, который сам пытается объединять страницы. Проблема в том, что когда ядро делает это (defrag), оно может "заморозить" процесс на сотни миллисекунд. Для Go-приложения с жесткими требованиями к задержкам (Low Latency) это фатально. **Staff рекомендация**: отключать THP (`madvise` mode) или использовать Explicit Huge Pages.

### Как Go 1.21+ улучшил работу с Huge Pages?
В новых версиях рантайм лучше выравнивает арены памяти по границам огромных страниц, что позволяет эффективнее использовать аппаратные оптимизации без ручной настройки ОС.

### Как проверить, есть ли промахи TLB в вашем Go сервисе?
Использовать инструмент `perf` в Linux: `perf stat -e dTLB-load-misses,iTLB-load-misses ./my-app`. Если количество миссов огромно, стоит задуматься о настройке Huge Pages.

### Накладные расходы на Huge Pages
При использовании 2MB страниц, даже для объекта в 1 байт система может выделить всю страницу (зависит от настроек аллокатора). Это может привести к резкому росту RSS (Resident Set Size) памяти. В облачных средах с жесткими лимитами (Overcommit) это может привести к выселению пода из ноды.
