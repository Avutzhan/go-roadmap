# 1. CPU & Heap Profiling

## CPU Profiling
Записывает выборку (sampling) стеков вызовов каждые 10мс (или чаще).
- **Цель**: Найти "горячие" функции, которые потребляют больше всего времени процессора.
- **Инструмент**: `go tool pprof`.

## Heap Profiling (Память)
- **alloc_objects**: Сколько всего объектов было выделено.
- **inuse_space**: Сколько памяти занято прямо сейчас.
Помогает находить утечки и неэффективные аллокации.

## Вопросы для интервью (Senior/Staff)

### В чем разница между Heap Profile и Stack Trace?
Heap profile — это агрегированная статистика по аллокациям в куче. Stack trace (дамп горутин) — это моментальный снимок того, что и где выполняется прямо сейчас. Профиль памяти полезен для поиска утечек, дамп горутин — для поиска дедлоков или подвисших задач.

### Как найти утечку памяти с помощью pprof?
Нужно снять два профиля с интервалом (например, 30 минут) и сравнить их:
`go tool pprof -base profile1.pb.gz profile2.pb.gz`.
Те объекты, чье количество в `inuse_space` выросло и не уменьшилось, являются кандидатами на утечку.

### Почему CPU профайлер может "врать" (Sampling bias)?
Профайлер делает снимки только в определенных точках (например, при вызове функций или на границах стека). Если функция делает тяжелые вычисления внутри одного длинного цикла без вызовов других функций, она может не попасть в выборку или её вес будет искажен. Также профайлер не видит время ожидания ввода-вывода (IO wait).

### Как визуализировать профили?
Команда `(pprof) web` открывает интерактивный граф связей в браузере (требует установленного `graphviz`). Также современный стандарт — Flame Graphs (пламенные графики), которые позволяют быстро оценить иерархию вызовов.
