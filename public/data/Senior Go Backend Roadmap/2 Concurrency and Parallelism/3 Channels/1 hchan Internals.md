# 1. hchan Internals: Deep Dive

## Устройство структуры hchan
В Go канал — это не просто магическая труба, а конкретная структура данных в куче (`runtime/hchan`):

```go
type hchan struct {
    qcount   uint           // Текущее число элементов в буфере
    dataqsiz uint           // Вместимость буфера (make(chan T, size))
    buf      unsafe.Pointer // Указатель на массив элементов (циклический буфер)
    elemsize uint16
    closed   uint32
    elemtype *_type
    sendx    uint           // Индекс следующей записи
    recvx    uint           // Индекс следующего чтения
    recvq    waitq          // Очередь ожидающих читателей (sudog)
    sendq    waitq          // Очередь ожидающих писателей (sudog)
    lock     mutex          // Мьютекс для защиты всех вышеперечисленных полей
}
```

## Почему канал — это ссылка?
Когда вы создаете канал `make(chan int)`, рантайм выделяет `hchan` в куче и возвращает указатель. Поэтому передача канала в функции всегда передает ссылку на одну и ту же структуру данных.

## Вопросы для интервью (Senior/Staff)

### Что такое sudog и зачем он нужен в очередях recvq/sendq?
`sudog` — это абстракция над горутиной, находящейся в очереди ожидания. Мы не можем поместить саму горутину (G) прямо в несколько очередей, так как одна горутина может участвовать в `select` по нескольким каналам одновременно. `sudog` содержит указатели на горутину, на элемент данных и на следующий/предыдущий элементы очереди.

### Как работает запись в канал под капотом (алгоритм)?
1. **Lock**: Захват `hchan.lock`.
2. **Direct Send**: Если в `recvq` есть ожидающая горутина, данные копируются **напрямую** в ее стек (zero-copy), горутина разблокируется.
3. **Buffered Send**: Если в буфере есть место, данные записываются в `buf[sendx]`, индекс `sendx` инкрементируется.
4. **Block**: Если буфер полон и читателей нет, текущая горутина создает `sudog`, кладет его в `sendq` и уходит в спячку (`gopark`).
5. **Unlock**: Освобождение лока.

### Какова сложность операций с каналом?
Все атомарные операции (чтение/запись без блокировки горутины) имеют сложность **O(1)**. Однако, при высокой конкуренции (множество горутин на один канал) время ожидания мьютекса `hchan.lock` может расти, создавая бутылочное горлышко.

### Почему в структуре hchan используется mutex, если Go пропагандирует "Do not communicate by sharing memory"?
Это классический "парадокс". Для реализации высокоуровневого примитива передачи сообщений (каналов) рантайму внутри необходимы низкоуровневые примитивы синхронизации памяти (мьютексы). Это скрыто от пользователя для обеспечения безопасности и простоты.
