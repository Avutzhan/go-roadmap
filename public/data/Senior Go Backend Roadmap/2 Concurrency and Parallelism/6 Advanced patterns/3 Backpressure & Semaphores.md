# 3. Backpressure & Semaphores

## Semaphore Pattern
Механизм ограничения количества одновременных операций. В Go реализуется через буферизованные каналы:
```go
sem := make(chan struct{}, limit)
sem <- struct{}{} // захватить слот
go func() {
    defer func() { <-sem }() // освободить слот
    doWork()
}()
```
Также есть официальная реализация в `x/sync/semaphore`, поддерживающая взвешенные семафоры (захват сразу N слотов).

## Backpressure (Обратное давление)
Это стратегия защиты системы от перегрузки. Если сервис не успевает обрабатывать запросы, он должен "дать отпор", чтобы не упасть полностью.
**Способы реализации:**
1. Ограничение входящей очереди (буферизованный канал конечного размера).
2. Возврат ошибки (например, HTTP 503 или 429) при переполнении канала.
3. Сброс старых задач в пользу новых (LIFO-сброс).

## Вопросы для интервью (Senior/Staff)

### Чем семафор на каналах отличается от sync.Mutex?
Семафор позволяет зайти **М** горутинам, мьютекс — только **одной**. Также мьютекс имеет понятие владения (только тот, кто залочил, может разлочить), а семафор на каналах просто управляет счетчиком заполненности.

### Как реализовать Load Shedding (сброс нагрузки) на каналах?
Использовать `select` с блоком `default`:
```go
select {
case queue <- task:
    // задача принята
default:
    // очередь полна, возвращаем 503/ошибку сразу
}
```
Это предотвращает накопление огромной очереди в памяти и рост задержек (latency spike) для всех пользователей.

### В чем разница между Throttling и Rate Limiting?
- **Rate Limiting**: Ограничение количества запросов за единицу времени (например, 100 запросов в минуту).
- **Throttling**: Замедление обработки запросов при достижении порога. В Go это можно реализовать через `time.Sleep` или ограничение пула горутин.

### Что такое "In-flight requests" и почему эта метрика важна?
Это количество запросов, которые уже приняты в работу процессом, но еще не завершены. Мониторинг "In-flight" горутин позволяет понять, справляется ли сервис с нагрузкой. Если количество CPU стабильно, а "In-flight" растет — значит, сервис заблокирован на внешних ресурсах (БД, API, диск).
