# 2. WaitGroup & Once

## sync.WaitGroup
Примитив для ожидания группы горутин. Содержит 64-битный счетчик, разделенный на:
1. Количество активных горутин (counter).
2. Количество горутин, ждущих на `Wait()` (waiter count).
3. Семафор для пробуждения.

### Критические правила
- `Add()` должен вызываться **до** запуска горутины в родительском потоке.
- Количество вызовов `Done()` должно строго соответствовать значению в `Add()`.
- Нельзя переиспользовать `WaitGroup`, пока предыдущий `Wait()` не завершился.

## sync.Once
Гарантирует, что функция `f` выполнится ровно один раз, даже если `Do(f)` вызывается из тысяч горутин одновременно.

### Реализация (Staff уровень)
```go
if atomic.LoadUint32(&o.done) == 0 { // Fast path
    o.doSlow(f) // Slow path с мьютексом
}
```
`sync.Once` использует двойную проверку: сначала дешевый атомарный `Load`, и только если он 0 — захватывает мьютекс и выполняет функцию. Это делает последующие вызовы практически бесплатными.

## Вопросы для интервью (Senior/Staff)

### Можно ли передавать WaitGroup по значению в функцию?
Нет. Это скопирует состояние счетчика. Если вызвать `Done()` на копии, это не уменьшит счетчик оригинала, и `Wait()` зависнет навсегда. Всегда передавайте через указатель.

### Что будет, если счетчик WaitGroup уйдет в минус?
Произойдет мгновенная паника: `sync: negative WaitGroup counter`. Это типичная ошибка, когда `Done()` вызывается больше раз, чем было указано в `Add()`.

### Гарантирует ли sync.Once завершение функции f для всех горутин?
Да. Если несколько горутин вызвали `Once.Do(f)` одновременно, одна выполнит `f`, а остальные будут **заблокированы** на мьютексе внутри `Once`, пока `f` не завершится. Только после этого они все разблокируются и пойдут дальше. Это предотвращает использование недоинициализированных ресурсов.

### Как реализовать сбрасываемый sync.Once (Reset)?
В стандартной библиотеке такого нет. Это антипаттерн, так как нарушает гарантию "выполнится только один раз за всю жизнь объекта". Если вам нужно повторное выполнение, используйте `sync.Mutex` или атомарные флаги.
